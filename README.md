The instances rely on a Serializable file of PortNumbers class (defined in the program) to communicate their port numbers to each other by creating the file with a newly instance of PortNumbers to establish a port number reservation system, deserializing and accessing the object to confirm that a user chosen port number that is prompted at startup time isn't taken, and serializing with updated port number to reserve an untaken port number. If the program terminates normally, it'll deserialize and serialize to remove its reservation, but if not, then the shutdown hook that was defined by a method invoked from Java's Runtime package will automatically deserialize and serialize to remove its reservation.

PortNumbers consist of 4 data members. 2 to hold reserved port numbers, and 2 others to allow an instance to determine whether to reserve on the first or second reservation slot, with the second slot being used if the instance is the second to reserve.
